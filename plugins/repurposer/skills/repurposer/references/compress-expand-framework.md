# Compress & Expand Framework

## The Core Idea

Every idea exists on a spectrum. A newsletter section is one position on that spectrum — an expanded version of a core insight. Short-form content is the compressed version. The same idea can live at any point on the spectrum. You don't write new content; you change the compression level.

Think of it like an accordion: push it in and you get a one-liner. Pull it out and you get a full newsletter section. Same notes, different positions.

## Compress: Long List → Short List

When a newsletter contains a list of 7–12 items, the compression move is to select 2–4 and let the shorter list carry more explanation per item.

**Before (10-item list, one line each):**
> 10 ways to write newsletters on any topic: tips, steps, lessons, examples, frameworks, templates, comparisons, stories, checklists, predictions.

**After (3-item list, one sentence each):**
> 3 ways to write newsletters on any topic:
> 1. Tips — tell them how to do something better
> 2. Steps — show them the order to do it in
> 3. Lessons — explain why it matters

Same idea. Fewer items. More depth per item. Easier to absorb.

**The rule:** As list length decreases, explanation per item can increase proportionally. A 10-item list needs ultra-compressed labels. A 3-item list can carry a full sentence per item.

## Expand: One-Liner → Full Post

The expansion move is the reverse: take a compressed claim or list item and unpack it into a standalone post.

**Before (one item from a list):**
> Lessons — share your reflections

**After (expanded into a standalone note):**
> The most underused newsletter format is the lesson.
>
> Not a tip (what to do). Not a step (how to do it).
>
> A lesson answers: why does this actually matter?
>
> Most writers skip it because it requires vulnerability — admitting what you got wrong, what surprised you, what changed your mind.
>
> That's exactly why it works.

**The rule:** Any list item that has a story, a personal example, or a counterintuitive angle behind it is a candidate for expansion.

## Practical Application

### When to Compress

Use compression when:
- The newsletter section is a long list
- You're targeting platforms with short format limits (Substack Notes, Twitter)
- The core value is in the structure, not the detail
- The reader needs to absorb it at a glance

### When to Expand

Use expansion when:
- A single idea from the newsletter is strong enough to carry a full post
- You have a personal story or example that pairs with one item
- The compressed version feels thin or generic
- You're targeting LinkedIn, where longer posts with narrative perform better

## The Infinite Version Principle

An idea doesn't have one correct format. The same insight can produce:
- A 280-character tweet
- A 3-item Substack note
- A 10-item list post
- A paragraph story post
- A 5-tweet thread
- A LinkedIn post with a hook, 3 paragraphs, and a CTA

All five are the same idea at different positions on the spectrum. Once you have one, you can derive all the others.

## Application in the Repurposer Skill

When processing a newsletter:
1. Identify all lists — these are compression candidates
2. Identify all bold/key statements — these are expansion seeds
3. For each list: generate the compressed version (pick best 2–4 items)
4. For each strong statement: generate the expanded version (add story, stakes, or contrast)
5. Label each output by type: compressed list / expanded narrative / hook / thread seed
